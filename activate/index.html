Activation flow

Generate IPR Create canonical identity record
Receive receipt Hash-based cryptographic proof
Verify ex-ante PASS / FAIL deterministic check

Operational notes

The activation process does not create a central identity database. The system is designed to operate with minimal data exposure.

Receipts contain cryptographic references only. Verification is deterministic and publicly reproducible.

If verification cannot be completed, result is considered invalid (fail-closed).
