Minimal fail-closed chain — Phase 1

Minimal chain (fail-closed)

The EU Pilot is implemented as a minimal chain. If any required evidence is missing, the system must not proceed.

Event → HRR → DGV → Certified Time & Trace → Long-Term Evidence Vault
Fail-closed rule: uncertainty is treated as risk. Outcome must be FAIL_CLOSED.

Core components

Component 01

Human Root Reference (HRR)

Technical human attribution for critical actions. Ensures there is a real, accountable human behind the request.

  • unique per individual
  • not duplicable
  • auditable
  • suspendable (not erasable)
HRR is not a civil identity. It is a technical root of accountability.
Component 02

Decision Gate Validator (DGV)

Validates or blocks actions before execution, enforcing standard outcomes.

  • mandatory fail-closed behavior
  • standard outcomes: ALLOW / REVIEW / DENY / FAIL_CLOSED
  • machine-readable logs
FAIL is protection, not failure. Execution without validation is non-conformant.
Component 03

Certified Time & Trace

Metrologically valid time and traceability. What cannot be measured and traced is not enforceable.

  • trusted timestamps with known error bounds
  • traceability and reproducible verification
  • consistent reference time source
Component 04

Long-Term Evidence Vault

Long-term evidence preservation designed to remain verifiable for decades.

  • retention ≥ 20 years
  • offline verification
  • periodic re-timestamping
  • ETSI / eIDAS compatible LTA profiles
