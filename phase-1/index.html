Fase 1 (ONLY)

Livello operativo minimo per cybersecurity UE. Qui si implementa solo la catena di attribuzione, validazione e prova. Nessun “sistema identità”, nessuna custodia dati, nessuna promessa bancaria.

Fail-closed: se manca un anello (HRR, DGV o LTA), l’atto non nasce. Il blocco è una protezione.
append-only hash-only GDPR-min risk-based

Catena minima

Struttura operativa:

HRR  →  DGV  →  LTA Vault
  • HRR (Human Root Reference): attribuzione umana/pseudonima dell’atto.
  • DGV (Decision Gate Validator): controlli di policy e integrità (ALLOW/REVIEW/DENY/FAIL-CLOSED).
  • LTA (Long-Term Archive): prova durevole e verificabile nel tempo.

Regole operative

  • Se manca HRR → FAIL-CLOSED.
  • Se mancano integrità o requisiti minimi → FAIL-CLOSED.
  • Se manca prova durevole (LTA) → l’atto resta non opponibile.
  • La verifica tecnica espone solo esiti: VALID / INVALID / INCONCLUSIVE.
