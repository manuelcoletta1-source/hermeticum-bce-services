JOKER Gate (Decision Gate)

Il Gate verifica ex-ante e restituisce: PASS oppure FAIL + reason_code. In caso di incertezza → FAIL (fail-closed).

Input minimo

  • ipr_id
  • job_hash
  • context_hash

Output

  • PASS + permit (token di esecuzione)
  • FAIL + reason_code

Il datacenter esegue solo se il permit è valido e coerente con gli hash.
