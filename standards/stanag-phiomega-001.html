STANAG–ΦΩ–001

Allegato strutturale a STANDARD UE–ΦΩ–001. Definisce requisiti tecnici minimi per implementazione e audit.

structural implementation audit hash-only fail-closed
Documento tecnico. Non normativo primario. Non onboarding.

Metadati

ID: STANAG-PHIOMEGA-001
Versione: 1.0
Relazione: Allegato a STANDARD UE–ΦΩ–001
Stato: Pubblico (read-only)
Ambito: Implementazione / Audit

Scopo

  • definire requisiti minimi verificabili
  • ridurre discrezionalità implementativa
  • abilitare audit tecnico riproducibile

Requisiti obbligatori (MUST)

  • Parsing JSON deterministico
  • Validazione schema minima
  • Fail-closed su campo mancante
  • Hash SHA-256 (hex, 64 char)
  • Timestamp ISO-8601 (UTC)

Requisiti consigliati (SHOULD)

  • Verifica locale (no server)
  • Separazione UI / logica
  • Output machine-readable

Schema minimo (estratto)

{
  "HRR": "string",
  "POLICY": "string",
  "PAYLOAD_MIN": {
    "desc": "string",
    "content_hash_sha256": "string|null",
    "timestamp": "string|null",
    "created_utc": "string"
  }
}

Ogni campo mancante rende l’evento INVALID.

Esiti di verifica

  • VALID: struttura coerente
  • INVALID: violazione requisito
  • INCONCLUSIVE: dati insufficienti per confronto hash

Sicurezza

  • Nessuna custodia di payload
  • Nessun identificatore personale obbligatorio
  • Hash come unico legame al contenuto

Error handling

Gli errori devono essere espliciti, deterministici e riproducibili. Nessun fallback “best effort”.

Relazioni

  • STANDARD UE–ΦΩ–001
  • Fase 1 (ONLY)
  • Verify

Audit

Un audit tecnico deve poter riprodurre:

  1. parsing
  2. validazione schema
  3. calcolo hash
  4. confronto deterministico

STANAG–ΦΩ–001 è un allegato tecnico: riduce ambiguità e aumenta verificabilità.
