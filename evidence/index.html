Evidence / Verification

Evidenze e verifica tecnica

Questa pagina descrive come verificare integrità e coerenza di un IPR in modo auditabile. È una guida per cittadini avanzati, tecnici, revisori e contesti forensi.

Definizioni operative

Manifest

Documento strutturato che descrive un IPR e i suoi riferimenti (metadati, versioni, hash).

Hash

Impronta crittografica di un contenuto. Se cambia anche 1 carattere, l’hash cambia.

Evidenza

Insieme di elementi verificabili: file, hash, timestamp/anchor, log, riferimenti.

Cosa dimostra (e cosa no)

Dimostra

  • Integrità: il contenuto non è stato alterato
  • Coerenza: manifest ↔ hash ↔ file corrispondono
  • Esistenza nel tempo: se ancorato, “esisteva” a una certa data

NON dimostra automaticamente

  • Identità legale della persona
  • Autenticità giuridica equivalente a firma qualificata
  • Autorità pubblica o riconoscimento istituzionale

Nota importante

L’interpretazione forense/giuridica dipende dal contesto e dal perimetro probatorio.

Procedura di verifica (semplice)

  1. Recupera manifest e i file associati.
  2. Ricalcola l’hash dei file in locale.
  3. Confronta gli hash calcolati con quelli dichiarati nel manifest.
  4. Se presente, verifica l’anchor/timestamp associato.

Esempio (CLI) — SHA-256

# Linux / macOS
shasum -a 256 FILE.ext

# Windows PowerShell
Get-FileHash FILE.ext -Algorithm SHA256

Fail-closed (regola di sicurezza)

Se anche un solo controllo fallisce (hash diverso, manifest incoerente, file mancanti), la verifica deve essere considerata NON valida. Questo è il principio fail-closed: meglio fermarsi che accettare una prova dubbia.
