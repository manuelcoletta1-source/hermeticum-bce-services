Definizioni operative

Manifest

Documento strutturato che descrive un IPR: metadati, versioni, riferimenti e hash.

Hash

Impronta crittografica di un contenuto. Se cambia anche 1 bit, l’hash cambia.

Evidenza

Insieme verificabile: file + hash + (eventuali) timestamp/anchor + log/versioning.

Cosa dimostra (e cosa no)

Dimostra

  • Integrità: il contenuto non è stato alterato (hash match)
  • Coerenza: manifest ↔ hash ↔ file corrispondono
  • Esistenza nel tempo: se ancorato, “esisteva” a una certa data

NON dimostra automaticamente

  • Identità legale della persona
  • Autenticità giuridica equivalente a firma qualificata
  • Riconoscimento istituzionale automatico

Regola pratica

Se mancano dati o c’è mismatch: esito Invalid o Inconclusive. Non forzare mai “Valid”.

Procedura di verifica (audit-grade)

  1. Recupera manifest e file associati.
  2. Ricalcola gli hash in locale sui file.
  3. Confronta con gli hash dichiarati nel manifest.
  4. Se presente, verifica l’anchor/timestamp associato.
  5. Registra esito e conserva evidenze/versioni.

Esempio CLI — SHA-256

# Linux / macOS
shasum -a 256 FILE.ext

# Windows PowerShell
Get-FileHash FILE.ext -Algorithm SHA256

Fail-closed

Principio operativo: se anche un solo controllo fallisce (hash diverso, file mancanti, manifest incoerente), la verifica deve essere considerata NON valida.

Esiti standard

Valid

Match completo: manifest coerente, hash corretti, evidenze presenti.

Invalid

Mismatch o alterazione: hash differente, file non coerenti, evidenze rotte.

Inconclusive

Dati insufficienti: non si può concludere. Esito da trattare come non accettabile in audit.

Buona pratica di conservazione

  • Conserva manifest e file originali
  • Conserva output di verifica (hash calcolati) con data/ora
  • Versiona modifiche e conserva changelog
  • Non sovrascrivere: aggiungi nuova traccia
