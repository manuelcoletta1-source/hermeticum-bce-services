IPR Technical Documentation

Reference documentation for the Identity Primary Record infrastructure. Designed for public verification, auditability and EU-aligned deployment.

Activate Verify Status

Core principles

Fail-Closed If verification fails → execution invalid
Audit-First Outputs designed for inspection
Hash-Only No raw personal data required
No Data Custody No centralized identity storage

System architecture

Layer 1 — Identity anchor
External or sovereign identity reference.

Layer 2 — Attribution
Responsibility binding and timestamp.

Layer 3 — Evidence
Hash-based receipt and append-only trace.

Layer 4 — Verification
Deterministic PASS / FAIL.

Layer 5 — Interoperability
B2B · B2G · EU pilot-ready.

Verification model

Input: receipt JSON or SHA-256 reference
Process: deterministic hash verification
Output: AUTHORIZED or DENIED
Mode: ex-ante (before execution)
