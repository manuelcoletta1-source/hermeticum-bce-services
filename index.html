IPR: una radice tecnica verificabile (non un’identità legale)

Qui si usa l’IPR (Identity Primary Record) per creare continuità tracciabile tra te, i tuoi file e i tuoi processi: un manifest + hash che descrivono “questa cosa esisteva così”, verificabile in audit. La fiducia non è narrativa: nasce dalla verifica. In caso di dubbio → fail-closed.

Perimetro obbligatorio: IPR non è SPID/CIE/eIDAS, non è firma elettronica qualificata, non produce effetti giuridici automatici.

La pipeline (in 60 secondi)

Questa è la sequenza minima per ottenere un record “auditabile”.

1) CreateGeneri manifest + hash (radice tecnica)
2) OutputSalvi manifest/evidenze: senza conservazione non c’è continuità
3) VerifyRicalcolo hash → esito Valid / Invalid / Inconclusive
4) EvidenceCostruisci il “pacchetto prova”: file + hash + log/versioning
Vai a Create Vai a Output Vai a Verify Vai a Evidence

Cosa ottieni (in pratica)

  • Integrità verificabile: se cambia anche 1 bit, lo vedi (hash mismatch).
  • Coerenza: manifest ↔ file ↔ evidenze devono combaciare, altrimenti fail-closed.
  • Continuità: evoluzione per versioni/derivazioni, senza cancellare il passato.
  • Auditabilità: procedura ripetibile (utile per contesti tecnici e forensi).

Non equivale a (perimetro chiaro)

  • Identità legale UE (SPID/CIE/eIDAS/EUDI Wallet)
  • Firma elettronica qualificata / servizio fiduciario qualificato
  • Riconoscimento istituzionale automatico
