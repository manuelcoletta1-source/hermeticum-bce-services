Questo è un gateway digitale UE per rilasciare un pacchetto di riconoscimento minimo (IPR) e una receipt firmata ED25519, con possibilità di ancoraggio (BTC/ETH/IPFS) e creazione di un evidence pack verificabile.

Nessuna custodia: il portale non conserva chiavi, non trattiene documenti, non fa da identity provider. Modello: GDPR-min, hash-only, audit-first, fail-closed.

Percorso rapido (4 step)

Se fai solo questi quattro passaggi, ottieni un set coerente e verificabile. Ogni step è client-side. Nessun upload.

  1. 1) Create — genera IPR + receipt firmata
    Crea ipr.json (hash-only) e receipt.json con firma ED25519. Opzionale: SIGNING_KEYPACK.json cifrato per continuità firma pack.
    Apri Create
  2. 2) Verify — verifica locale (fail-closed)
    Verifica hash + firma ED25519. Controlla anchors (se presenti). Opzionale: verifica PACK_MANIFEST.json.
    Apri Verify
  3. 3) Anchor — costruisci “anchor pack” (manuale)
    Inserisci riferimenti a BTC/ETH/IPFS (se li hai) e genera un anchor-pack.json ordinato. Nessuna promessa: è un builder, non una rete di timestamping.
    Apri Anchor
  4. 4) Evidence Pack — crea PACK_MANIFEST firmato
    Costruisci un pack verificabile: IPR + receipt + anchor-pack + README proof + PACK_MANIFEST.json firmato.
    Apri Evidence Pack

Nota operativa UE: questo portale non sostituisce eIDAS/EUDI Wallet. È un gateway sperimentale “audit-first” per pacchetti verificabili e minimali.

Security model (GDPR-min by design)

  • Nessun upload: i file restano sul dispositivo.
  • Hash-only: non serve esporre dati personali per verificare integrità.
  • ED25519: receipt e manifest possono essere firmati localmente.
  • Fail-closed: se qualcosa non torna, la verifica fallisce (nessuna “tolleranza”).
  • No custody: nessuna gestione chiavi/documenti, nessun ruolo da Identity Provider.
