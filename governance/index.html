Evidence Pack (catena minima di prova)

Un Evidence Pack è un pacchetto di elementi che permette a un terzo di verificare una cosa semplice: questo contenuto corrisponde a questo hash, e questo hash è riferito da un manifest IPR. Nessuna promessa legale, nessuna autorità nascosta: solo verificabilità ripetibile.

Cos’è (definizione operativa)

Evidence Pack = file + hash locale + manifest IPR + report di verifica + (opzionale) log/versioning. Se manca un pezzo critico, il pacchetto non è “quasi valido”: è incompleto.

Tre livelli (pratici)

Non sono “certificazioni”. Sono livelli di completezza della prova tecnica.

  • BASE — conservazione minima + integrità verificabile.
  • STRONG — verifica ripetibile + report + versioning.
  • ENTERPRISE — chain-of-custody + ruoli + policy conservazione + audit trail.

Struttura cartella standard (consigliata)

Nome cartella suggerito: evidence-YYYYMMDD-HHMM-ipr_id

evidence-pack/
  artifacts/
    original-file.ext
  hashes/
    sha256.txt
  ipr/
    IPR-manifest.json
    IPR-verify-report.json
  meta/
    README.txt
    changelog.txt

Regola: ciò che sta in artifacts/ è “materiale”. Il resto sono tracce di controllo.

Procedura (ripetibile)

Obiettivo: rendere possibile a un terzo di rifare i calcoli senza fidarsi di te.

Step 1 — Hash locale del file (audit-grade)

Esempi:

Windows (PowerShell):
  Get-FileHash .\artifacts\original-file.ext -Algorithm SHA256

Linux/macOS:
  shasum -a 256 ./artifacts/original-file.ext

Step 2 — Salva l’hash in hashes/sha256.txt

SHA-256: <hex-64>
FILE: artifacts/original-file.ext
GENERATED_AT_UTC: <YYYY-MM-DDTHH:MM:SSZ>
TOOL: <PowerShell Get-FileHash | shasum>

Step 3 — Inserisci il manifest IPR (da Create/Output)

Salva il manifest in ipr/IPR-manifest.json. Se il manifest è basato su testo incollato (Create UI), mantieni coerente l’oggetto verificato: per file binari, usa sempre l’hash locale come riferimento primario nel pack.

Step 4 — Verifica e salva il report

Esegui Verify e salva il report in ipr/IPR-verify-report.json. Se manca il contenuto → esito tipico: Inconclusive.

Step 5 — Meta (spiegazione asciutta)

README.txt (minimo):
- Che cosa contiene artifacts/
- Quale file è stato hashato
- Dove sta l’hash (sha256.txt)
- Dove sta il manifest IPR
- Dove sta il report Verify
- Regola fail-closed: se mismatch → Invalid

Regole fail-closed (semplici)

  • Se l’hash ricalcolato ≠ hash salvato → il pack è Invalid.
  • Se manca il file o manca l’hash → pack Incompleto.
  • Se il manifest non contiene hash leggibile → Invalid.
  • Se c’è solo struttura, senza contenuto verificabile → Inconclusive (non usare come prova forte).
