Evidence-first — Minimal audit workflow

What is being verified

Verification targets an Evidence Bundle: a minimal package that proves who requested what, when, and under which constraints, in a way that remains checkable over time.

This page can be used with pseudonymous identifiers. No personal data is required for public verification.

Evidence Bundle (minimal)

The pilot bundle is described by a machine-readable manifest and integrity proofs.

Evidence Bundle - manifest.json (hashes + references) - attachments (optional, may remain private) - timestamp reference (if present) - anchors (optional: LTA/IPFS/ledger)
Minimal publishing rule: you can publish only manifest + hashes publicly, while keeping attachments private, as long as hashes are verifiable.

Verification steps

  1. Compute hashes of the published files and compare with the manifest.
  2. Verify timestamp (if present): it must bind to the same hash.
  3. Confirm DGV outcome: if DENY or FAIL_CLOSED, the action must not have executed.
If hashes do not match, or timestamp verification fails, the bundle is not enforceable.

Local hash commands

Linux / macOS: shasum -a 512 manifest.json shasum -a 512 file.ext Windows (PowerShell): Get-FileHash .\manifest.json -Algorithm SHA512 Get-FileHash .\file.ext -Algorithm SHA512
If the environment cannot compute hashes reliably, verification must be considered incomplete.
