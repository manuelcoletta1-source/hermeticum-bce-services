Il sistema è binario: o è verificabile, o non è eseguibile

Questa pagina definisce il modello tecnico minimo: IPR (precedenza), un gate (AI JOKER-C2) che fa pre-check, e un criterio di uscita: PASS/FAIL. In questo mondo non esiste “quasi”: una differenza minima cambia l’hash e quindi cambia l’atto.

Definizioni minime

  • IPR: record primario che definisce precedenza, scope, policy e contesto.
  • Canon: serializzazione stabile del record (stesso input → stesso hash).
  • Digest: hash SHA-512 del canonico (identità tecnica del contenuto).
  • Gate ex-ante: componente che verifica prima dell’esecuzione.
  • Fail-closed: in dubbio, blocca. Mai “fallback silenzioso”.

Regola “virgola = FAIL”

Un hash è sensibile a ogni variazione. Questo è il punto: se cambia anche un carattere, cambia l’atto. Non è un bug, è la sicurezza.

virgola=FAIL append-only

Perché serve in AI (punto di rottura)

Senza un gate, molte AI operano in modalità fail-open: se il contesto è ambiguo, provano comunque. Con IPR + gate, l’ambiguità diventa blocco.

fail-open fail-closed

Interfaccia del gate (minima, implementabile)

INPUT:
  action_request            // cosa si vuole fare
  ipr_reference             // puntatore a canon+digest (e receipt se presente)
  context_fingerprint       // fingerprint del runtime (modello, tool, ambiente)
  policy_fingerprint        // fingerprint policy attiva

CHECK (fail-closed):
  if !verify(ipr_reference)             -> FAIL
  if context_fingerprint != ipr.context -> FAIL
  if policy_fingerprint  != ipr.policy  -> FAIL
  if violates(ipr.limits)               -> FAIL

OUTPUT:
  PASS | FAIL  // nessun output esecutivo se FAIL

IPR dentro la tua AI: cosa cambia

Senza IPR, la tua AI è “una macchina di output”: riceve input, produce risposta, e spesso tenta di completare anche quando manca evidenza. Con IPR, la tua AI diventa una estensione tecnica e legale del tuo perimetro: prima di agire deve dimostrare coerenza con policy e contesto.

  • AI domestica: comandi sensibili (pagamenti, accessi, domotica) solo con PASS.
  • AI professionale: strumenti e dati solo nel perimetro definito (no leak).
  • AI in flotte/swarm: comandi solo se precedenza verificata.
  • AI spazio/satelliti: control-plane evidence-locked (errore = FAIL).

Compatibilità UE (principio)

Questo modello è “UE-first” nel senso tecnico: minimizza dati (hash-only), rende verificabile l’atto (audit-first) e impedisce esecuzione senza evidenza (fail-closed).

GDPR-min audit-first fail-closed

Limite fondamentale

Un gate non “rende buone” le decisioni: rende misurabile se erano eseguibili. La responsabilità resta umana. Il sistema impedisce solo la fuga nel non verificabile.
