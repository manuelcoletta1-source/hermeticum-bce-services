Identity Primary Record (IPR)

Technical Standard · EU-aligned · Hash-based · No data custody

1. Definition

Identity Primary Record (IPR) is a cryptographic responsibility record generated by a biological or artificial subject. It establishes a verifiable temporal reference without storing personal data on external servers.

2. Generation

An IPR is generated client-side through deterministic cryptographic hashing (SHA-256). Input may include self-declared identity data and/or user-provided content. No personal data is transmitted or stored by the infrastructure.

3. Structure

Each IPR contains:
• Timestamp (UTC)
• Self-declared subject identity (optional)
• Cryptographic hash reference
• Algorithm specification
• Infrastructure seal (Hermeticum)

4. Verification

IPR integrity is verified by recalculating the cryptographic hash and comparing it to the receipt value. Verification is executable by any independent third party.

5. Audit Chain

Each IPR can generate subsequent audit records linked through previous receipt hashes. This creates a continuous, verifiable temporal chain of responsibility.

6. Data Custody

The system operates under a strict no-data-custody model. All data remains under user control. Hermeticum infrastructure does not store or process personal identity data server-side.

7. Compliance Orientation

Designed for compatibility with EU digital identity and GDPR principles:
• Data minimisation
• Self-declared identity
• Client-side cryptography
• Verifiable auditability
