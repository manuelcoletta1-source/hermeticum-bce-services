Onboarding operativo

Questa sezione descrive il flusso minimo “fail-closed” per entrare nel perimetro ΦΩ · Fase 1 senza introdurre custodia dati.

Obiettivo

  • Definire un soggetto responsabile (ruolo umano).
  • Stabilire un Decision Gate che non esegue senza prova.
  • Produrre un “evidence pack” verificabile nel tempo (hash-only).

Flusso minimo (Fase 1)

  1. Identifica il contesto: processo, sistema, rischio, confini.
  2. Nomina la responsabilità: chi può “pagare il costo” dell’atto.
  3. Configura il gate: regole, input richiesti, condizione di blocco.
  4. Definisci l’evidenza: cosa viene hashato, dove si conserva, come si verifica.
  5. Test fail-closed: prova l’assenza di prova → nessuna esecuzione.

Output

  • Policy dichiarata (UE-first · audit-first · fail-closed · hash-only)
  • Traccia tecnica riproducibile (verifica deterministica)
  • Nessuna custodia di dati personali (solo impronte / hash)
