# Create — Creare una traccia verificabile

Questa pagina spiega **cosa significa creare**
una traccia nel campo biocibernetico Hermeticum B.C.E.

Creare non è obbligatorio.
Creare è **irreversibile**.

---

## Cosa significa “creare”

Creare significa:
- produrre un output minimo (hash / receipt / manifest)
- fissarlo nel tempo
- renderlo verificabile

Creare **non significa**:
- registrarsi
- ottenere diritti
- essere riconosciuti da un’autorità
- attivare effetti legali automatici

---

## Prima di creare (guardrail)

Se una di queste condizioni non è chiara, **non creare**:

- accetti che la traccia resti nel tempo
- sai cosa stai fissando
- sai che non potrai cancellarla
- sai che puoi solo **aggiungere eventi successivi**

Se hai dubbi, fermati.

---

## Cosa viene creato (tecnicamente)

Il sistema produce:
- hash del contenuto
- metadati minimi
- eventuale firma crittografica

Il sistema **non conserva**:
- documenti originali
- dati personali
- contenuti sensibili

Tutto ciò resta **fuori** dal gateway.

---

## Tipi di output

Puoi creare:
- una ricevuta di esistenza (timestamp + hash)
- un evento append-only
- un evidence pack derivato

Ogni output è:
- verificabile offline
- indipendente dalla piattaforma
- riproducibile

---

## Responsabilità

Chi crea:
- resta responsabile del contenuto originale
- resta responsabile del contesto
- resta responsabile dell’uso futuro

Il sistema:
- non interpreta
- non giudica
- non protegge

---

## Se qualcosa va storto

Questo sistema è **fail-closed**.

- se manca un dato → output invalido
- se una firma non verifica → output invalido
- se un passaggio è ambiguo → output invalido

Meglio **non creare** che creare male.

---

## Dopo la creazione

Dopo aver creato puoi:
- verificare
- aggiungere un evento successivo
- raccogliere prove nel tempo

Non puoi:
- cancellare
- riscrivere
- negare l’esistenza della traccia

---

## Nota finale

Creare è un atto di **assunzione di continuità**.

Se non vuoi continuità,
questo non è il posto giusto.
