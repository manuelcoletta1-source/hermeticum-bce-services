Standards

Questa sezione definisce requisiti, limiti e criteri per uso B2B/B2G. Non è un atto di autorità: è documentazione riproducibile per implementazione e audit.

← Torna a Tech ← Gateway Verify

Cosa è uno “standard” qui

  • Un set di requisiti minimi (UE-first) per garantire verificabilità e riproducibilità.
  • Un perimetro di limiti dichiarati (cosa il sistema non fa).
  • Una base per audit: ciò che non passa i controlli non deve procedere (fail-closed).

Cosa NON è

  • Non è un riconoscimento legale automatico.
  • Non sostituisce eIDAS / EUDI Wallet.
  • Non è una promessa di validità “universale”.

Standard candidato (B2G)

Specifica candidata per governance: adozione istituzionale, requisiti di opponibilità e criteri di conformità.
