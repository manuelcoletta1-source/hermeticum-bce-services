1. Obiettivo di sicurezza

L’obiettivo del servizio è permettere all’utente di generare prove verificabili (es. hash) e bundle locali per integrità e coerenza dei dati, riducendo al minimo il rischio di esposizione.

2. Cosa fa (e cosa non fa)

FA:

  • calcolo di hash (es. SHA-512) lato browser
  • generazione di file/bundle locali scaricati dall’utente
  • verifica di integrità tramite confronto hash

NON FA:

  • non è una certificazione ufficiale UE
  • non sostituisce documenti statali (CIE, passaporto, ecc.)
  • non protegge se il dispositivo dell’utente è compromesso

3. Modello di minaccia (threat model)

Questo sistema è progettato per ridurre i rischi più comuni, ma assume alcune condizioni:

  • Assunzione A: il browser non è alterato (estensioni malevole, malware, man-in-the-browser)
  • Assunzione B: l’utente controlla i file che seleziona e conserva i bundle in modo sicuro
  • Assunzione C: l’utente usa una connessione affidabile (anche se il processo è locale)

Limite critico: se il dispositivo o il browser sono compromessi, qualunque misura “client-side” può essere aggirata. In quel caso la sicurezza reale dipende dalla bonifica del sistema, non dal sito.

4. Verifica offline (procedura consigliata)

Per verificare integrità e coerenza, confronta l’hash calcolato in due modi indipendenti:

  • hash calcolato sul sito (client-side)
  • hash calcolato localmente sul tuo sistema operativo

Esempi (indicativi):

# Linux / macOS
shasum -a 512 FILE.pdf

# Windows PowerShell
Get-FileHash -Algorithm SHA512 .\FILE.pdf

5. Privacy e assenza di archiviazione

Il servizio è progettato per funzionare senza inviare o archiviare dati su server: dati e file restano nel browser e sul dispositivo dell’utente.

6. Responsabilità operativa dell’utente

L’utente è responsabile di:

  • proteggere il proprio dispositivo (password, cifratura, aggiornamenti)
  • conservare i bundle in modo sicuro
  • non condividere file sensibili con terzi non autorizzati

7. Nota UE-first

“UE-first” indica un orientamento progettuale (GDPR, auditabilità, minimizzazione), non un riconoscimento pubblico. Non esiste alcuna approvazione o certificazione ufficiale UE implicita nel servizio.
